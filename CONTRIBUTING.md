# Contributing Guidelines

Thank you for your interest in contributing to this project.

## How to Contribute
You can contribute by:
- Reporting bugs
- Suggesting new features
- Improving documentation
- Submitting code changes
- Refining templates such as `Source.md` and `Feedback.md`

## Workflow
1. Fork the repository or create a feature branch
2. Make your changes in a separate branch
3. Write clear commit messages
4. Open a pull request with a clear description

## Branch Naming
Recommended branch naming:
- `feature/...`
- `fix/...`
- `docs/...`

Examples:
- `feature/source-parser`
- `fix/missing-ingredient-bug`
- `docs/update-readme`

## Pull Request Checklist
Before submitting a pull request:
- [ ] Code runs correctly
- [ ] Documentation is updated if needed
- [ ] Changes are focused and minimal
- [ ] Commit messages are clear
- [ ] Related issues are referenced if applicable

## Coding Style
- Keep functions small and readable
- Use clear variable and file names
- Add comments when the logic is not immediately obvious
- Avoid unnecessary complexity

## Reporting Issues
When reporting an issue, please include:
- A clear title
- Expected behavior
- Actual behavior
- Steps to reproduce
- Relevant logs or screenshots if available

## Questions
For questions or discussions, please open an issue.
